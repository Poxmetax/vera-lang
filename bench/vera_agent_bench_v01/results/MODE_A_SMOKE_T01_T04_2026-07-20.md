# VeraAgentBench v0.1 — Mode A smoke, T01–T04 (2026-07-20)

**Scope:** Mode A (full local CLI) smoke for **T01–T04 only**. T05–T08 were not
run, read, or modified in this experiment.

**Rule:** claim-less. Exit codes and oracle substrings only. This is not a
leaderboard, not a model comparison, and carries no language-capability claim
beyond "these four oracles ran and reported the results below".

---

## Method

Each task was exercised **twice, independently**:

1. **Oracle layer** — `powershell -NoProfile -File bench/vera_agent_bench_v01/task_*/checks/check.ps1`,
   invoked from `vera-lang/`. The script's own exit is the verdict: `0` = PASS,
   `1` = FAIL.
2. **Raw CLI layer** — `cargo run -q -p vera -- [--prove] <task>/initial/main.vera`,
   run separately to capture the vera CLI's own exit code and stdout with no
   oracle in between.

Both layers agreed on all four tasks. The raw layer exists so the CLI exit code
is **recorded directly rather than inferred** from the oracle's PASS.

A baseline `cargo build -p vera` was run first (exit 0) so that any task FAIL
would be attributable to the language/oracle rather than to a broken build.

**The `Exit` column is the vera CLI exit code.** All four oracle scripts exited
`0` (PASS) — those two numbers are different things and are reported separately
on purpose.

**T03 is the one row where a non-zero CLI exit is the success condition.**
`task_T03_prove_refuted_false_ensures/checks/check.ps1:7` fails the task unless
the CLI exits exactly `3`. Reading the Exit column as "0 = good" would
misreport that row as a failure.

## Honest limits

- **Mode A, one host, four tasks.** Nothing here measures other hosts, other
  agents, or Mode B.
- **T05–T08 untested** in this run, by design.
- **No SKIP(env) row occurred** because cargo and Z3 both resolved on this host.
  On a host missing either, T02–T04 would be `SKIP(env)` — not FAIL.
- **The oracles are exit-code plus substring matches.** They confirm the CLI
  emits the expected label; they do not establish that the underlying prove
  result is semantically correct beyond that label.
- Z3 was located through the ambient PATH; the CLI reported
  `[SOFT-Z3-PATH] using Z3: z3` on each prove task.
